# Design System

三套主题预设，统一遵循「白底 + 沉稳色彩模块」的现代美学。每套预设定义完整的视觉语言——色彩、字体、布局节奏和组件风格。

## 核心设计哲学

**白色是画布，色彩是内容的载体。**

所有主题共享以下原则：

1. **白底为王**：内容页一律使用纯白或极浅灰（#FAFAFA / #F8FAFC）背景，让色彩模块成为视觉焦点
2. **色彩注入方式**：通过卡片顶边/左边色条、数据高亮色、图标色块来注入色彩——不是大面积色块背景
3. **沉稳不花哨**：文字主色为深灰/墨色（#1E293B），强调色饱和度适中、不刺眼
4. **卡片承载内容**：所有信息住在圆角卡片里，不是漂浮的文本块
5. **呼吸感**：元素之间留足空间，宁可少放内容也不要拥挤
6. **字号层次**：页标题 28-36px bold → 卡片标题 14-18px bold → 正文 11-14px → 注释 8-10px
7. **画布自适应**：支持 LAYOUT_WIDE / LAYOUT_16x9 / LAYOUT_16x10 / LAYOUT_4x3

## Theme 1: Executive (mckinsey)

**美学定位**：白底 + 深灰墨色权威 + 青绿/琥珀双强调。沉稳、专业、数据驱动。

**适用场景**：高管汇报、董事会演示、咨询报告、政策分析、企业战略。

### Color Tokens

| Token | Hex | 用途 |
|-------|-----|------|
| dominant | `#1E293B` | 墨灰色 — 标题、边框、主要文字强调 |
| dominantMid | `#334155` | 中灰 — 次级标题 |
| dominantLight | `#475569` | 浅灰 — 辅助元素 |
| support | `#0F766E` | 深青 — 卡片色条、分类标识 |
| accent | `#0D9488` | 青绿 — 高亮、时间线、强调线 |
| accentSecondary | `#D97706` | 琥珀 — 警示、第二强调色 |
| background | `#FFFFFF` | 纯白 — 页面背景 |
| surface | `#FFFFFF` | 白色 — 卡片背景 |
| surfaceAlt | `#F1F5F9` | 极浅灰 — 次级面板背景 |
| text | `#1E293B` | 墨灰 — 主文字 |
| muted | `#64748B` | 中灰 — 标签、注释 |
| line | `#E2E8F0` | 浅灰 — 分割线、边框 |

### 视觉特征

- 封面：白底或极浅灰底，大号墨色标题，青绿色强调线
- 内容页：白底，卡片带 4px 青绿/琥珀色顶边
- 步骤流程：卡片顶边颜色渐变 `#1E293B → #334155 → #0F766E → #0D9488 → #D97706`
- 数据卡片：大号数字用强调色，标签用灰色

## Theme 2: Modern Tech (dark-tech)

**美学定位**：白底 + 紫罗兰/蓝色双主调 + 多彩分类色。现代、精致、技术感强但不压抑。

**适用场景**：技术架构、数字化转型、研究提案、央企/政府科技项目、产品技术文档。

### Color Tokens

| Token | Hex | 用途 |
|-------|-----|------|
| dominant | `#18181B` | 近黑 — 标题、主要文字 |
| support | `#3F3F46` | 深灰 — 次级元素 |
| accent | `#8B5CF6` | 紫罗兰 — 主强调色、色条、高亮 |
| accentBlue | `#3B82F6` | 蓝色 — 第二分类色 |
| accentAmber | `#F59E0B` | 琥珀 — 第三分类色 |
| background | `#FAFAFA` | 极浅灰 — 页面背景 |
| surface | `#FFFFFF` | 白色 — 卡片背景 |
| surfaceMuted | `#F4F4F5` | 浅灰 — 次级面板 |
| text | `#18181B` | 近黑 — 主文字 |
| muted | `#71717A` | 中灰 — 标签、注释 |
| line | `#E4E4E7` | 浅灰 — 分割线 |

### 视觉特征

- 封面：白底或极浅灰底，紫罗兰色强调线，大号标题
- 内容页：白底卡片，4px 紫/蓝/琥珀色顶边区分分类
- 架构图：分层卡片，每层用不同强调色标识
- 数据面板：紫罗兰色大号数字 + 灰色标签

**适用场景**：产品发布、投资人演示、创业路演、培训材料、SaaS 产品介绍、营销方案。

## 主题选择指南

| 场景 | 推荐主题 | 原因 |
|------|----------|------|
| 高管/董事会/C-suite | mckinsey | 沉稳权威，数据驱动 |
| 政府/央企/研究提案 | dark-tech | 现代技术感，多分类色系统 |
| 创业/投资人/产品发布 | genspark | 清新现代，亲和力强 |
| 咨询/顾问报告 | mckinsey | 结构清晰，专业可信 |
| 技术架构/数字化 | dark-tech | 分层可视化，紫蓝色系 |
| 培训/教育/工作坊 | genspark | 轻松友好，卡片式学习 |
| ESG/可持续发展 | mckinsey | 青绿色系天然契合 |
| 销售/营销 | genspark | 活力、色彩丰富 |

如果用户未指定主题，根据以下推断：
- 受众类型（高管 → mckinsey，技术 → dark-tech，创业 → genspark）
- 主题内容（政策/战略 → mckinsey，架构/基础设施 → dark-tech，产品/市场 → genspark）
- 语气关键词（"专业/权威" → mckinsey，"科技/现代" → dark-tech，"清新/活力" → genspark）

## 自定义覆盖机制

在 deck-spec.json 中：

```json
{
  "themePreset": "mckinsey",
  "theme": {
    "accent": "#7C3AED",
    "accentSecondary": "#059669"
  }
}
```

覆盖规则：
- `themePreset` 可选。省略则只使用 `theme` 对象
- 设置了 `themePreset` 但 `theme` 为空，则使用完整预设
- 两者都设置时：预设 → 覆盖 → 最终值
- 未在预设或覆盖中出现的 token 使用渲染器内置默认值
